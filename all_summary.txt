                              PROJECT OVERVIEW
================================================================================

CareerLens VPRO is an AI-powered career matching system that:
- Analyzes CVs (DOCX format) and extracts structured data
- Searches for relevant job opportunities via RapidAPI
- Scores jobs based on CV skills vs job requirements using Gemini AI
- Generates PDF/DOCX reports with career analysis and salary predictions
- Provides personalized learning roadmaps for skill gaps
- Supports background job scheduling for continuous monitoring

Key Features:
✓ CV Parsing & Skill Extraction
✓ AI-Powered Job Matching & Scoring
✓ Salary Prediction (CatBoost ML Model)
✓ PDF/DOCX Report Generation
✓ Learning Roadmaps
✓ Background Job Scheduler
✓ User Preferences Storage
✓ Database Support (SQLite, PostgreSQL, MySQL)

================================================================================
                        KEY SERVICE MODULES
================================================================================

1. CV PARSER (app/services/cv_parser.py)
   - Reads DOCX files using python-docx
   - Extracts text and structure
   - Returns structured CV dictionary

2. CV EXTRACTOR (app/services/extractor.py)
   - Processes structured CV data
   - Extracts: name, email, skills, education, experience
   - Uses AI for intelligent extraction

3. JOB SEARCH (app/services/job_search.py)
   - Calls RapidAPI for job listings
   - Filters results by query
   - Returns top 3 matching jobs

4. SCORER (app/services/scorer.py)
   - Uses Google Gemini API for AI scoring
   - Compares CV skills vs job requirements
   - Generates match_summary, verdict, missing_skills
   - Returns scores 0-100

5. SALARY PREDICTOR (app/services/salary_predictor.py)
   - Loads CatBoost ML model from research/
   - Takes: job_title, industry, experience, education, location, remote_ratio
   - Returns: Predicted salary in JOD
   - Uses metadata for market classification

6. ROADMAP (app/services/roadmap.py)
   - Analyzes missing skills
   - Generates learning steps
   - Suggests resources and durations
   - Provides upskilling path

7. PDF GENERATOR (app/services/pdf_generator.py)
   - ReportLab-based PDF generation
   - Includes: Executive summary, skills analysis, job leaderboard
   - Reports salary predictions
   - Professional formatting with color coding

8. DOCX GENERATOR (app/services/docx_generator.py)
   - python-docx-based DOCX generation
   - Similar content to PDF
   - Editable Word document output
   - Includes salary predictions in job table

9. BACKGROUND SCHEDULER (app/services/background.py)
   - APScheduler for continuous job monitoring
   - Schedules user-specific job searches
   - Sends notifications when matches found
   - Updates database with new matches

================================================================================
                        DOCKER DEPLOYMENT
================================================================================

Multi-Stage Build:
  Stage 1 (Builder):
    - Python 3.11 slim base
    - Installs build dependencies
    - Creates virtual environment
    - Installs all Python packages
  
  Stage 2 (Runtime):
    - Python 3.11 slim base
    - Copies venv from builder
    - Reduces final image size
    - Runs as non-root user (appuser)

Startup Sequence:
  1. Docker builds image from Dockerfile
  2. docker-compose.yml orchestrates services
  3. Container startup runs docker-entrypoint.sh
  4. Entrypoint runs Alembic migrations
  5. Uvicorn starts API server on port 8000
  6. Health check validates endpoint every 30s

Volume Mounts:
  - ./user_preferences.json:/app/user_preferences.json
  - Persists user preferences between restarts

Health Check:
  - Endpoint: GET http://localhost:8000/health
  - Interval: 30s, Timeout: 10s, Retries: 3
  - Start grace period: 5s

Environment Variables in Container:
  - All variables from .env passed to container
  - API keys secured via environment
  - DATABASE_URL for connection string

================================================================================
                        WORKFLOW & DATA FLOW
================================================================================

Complete Job Matching Pipeline:

  1. USER UPLOADS CV (DOCX)
     └─> POST /ultimate-match or /generate-pdf or /generate-docx
  
  2. CV PARSING
     └─> cv_parser.py reads DOCX file
         └─> Returns structured content
  
  3. DATA EXTRACTION
     └─> extractor.py processes content
         └─> Extracts: name, skills, education, experience
  
  4. JOB SEARCH
     └─> job_search.py queries RapidAPI
         └─> Returns top 3 matching jobs
  
  5. AI SCORING (Multi-threaded)
     └─> For each job:
         ├─> scorer.py calls Gemini API
         ├─> Compares CV skills vs job description
         ├─> Generates match_score (0-100)
         ├─> Identifies missing_skills
         └─> Creates verdict (Apply Now/Apply With Caution/Skip)
  
  6. SALARY PREDICTION
     └─> For each job:
         ├─> salary_predictor.py calls CatBoost model
         ├─> Takes: job_title, industry, experience, education, location
         └─> Returns: predicted salary in JOD
  
  7. RANKING
     └─> Sort by raw_score descending
         └─> Top 3 returned
  
  8. LEARNING ROADMAP
     └─> roadmap.py analyzes top job
         └─> Identifies missing skills
         └─> Suggests learning path with resources
  
  9. REPORT GENERATION
     └─> pdf_generator.py or docx_generator.py
         ├─> Executive summary
         ├─> Skills analysis
         ├─> Job leaderboard WITH SALARY PREDICTIONS
         ├─> Top job analysis
         └─> Learning roadmap
  
  10. USER RECEIVES REPORT
      └─> PDF or DOCX with full analysis

Background Job Scheduling:
  1. User saves preferences via /save-user
  2. background.py schedules job search
  3. APScheduler runs at specified intervals
  4. New jobs searched and scored
  5. Notifications sent for matches > threshold
  6. Database updated with new SeenJob entries

================================================================================
                        SALARY PREDICTION MODEL
================================================================================

Location: app/research/careerlens_salary_model.cbm (CatBoost)

Input Features:
  1. job_title (str) - Position name
  2. industry (str) - Industry/sector
  3. experience (str) - Years of experience level
  4. education (str) - Education qualification
  5. location (str) - Job location code
  6. market_type (str) - "high_market" or "local_market"
  7. remote_ratio (int) - 0-100 remote percentage
  8. is_remote (bool) - Binary remote flag

Output:
  - Predicted salary in JOD (Jordanian Dinar)
  - Rounded to 2 decimal places

Training Data:
  - final_master_dataset.csv (historical salary data)
  - Market classifications in careerlens_metadata.pkl
  - High-market countries: US, UK, CA, DE

Integration Points:
  - Called in /ultimate-match endpoint
  - Called in /generate-pdf endpoint
  - Called in /generate-docx endpoint
  - Results displayed in job tables and reports

================================================================================
                        TROUBLESHOOTING
================================================================================

Common Issues:

1. "No jobs found from API"
   - Check RAPID_API_KEY is valid
   - Verify RapidAPI subscription active
   - Check API endpoint availability
   - Try different job_query parameter

2. "Gemini API error"
   - Verify OPENROUTER_API_KEY
   - Check OpenRouter account credits
   - Confirm model name is available
   - Check internet connection

3. "Database connection error"
   - Verify DATABASE_URL format
   - Check database is running
   - Confirm credentials
   - Review alembic migrations

4. "Docker build fails"
   - Clear Docker cache: docker system prune -a
   - Rebuild: docker-compose build --no-cache
   - Check Docker daemon running
   - Verify disk space available

5. "Port 8000 already in use"
   - Change port in docker-compose.yml
   - Or kill process: lsof -i :8000 | kill -9
